# project specifications

## Goal

Vast numbers of projects involve reinventing the wheel. What if we could offer a way to reduce or eliminate such duplication of effort? What pieces are needed for this?

## Needed pieces

- [ ] council
- [ ] maturity assessment
- [ ] api request / evaluate process
- [ ] apportioning small pieces (promise deliverables)
- [ ] guidance for projects in trouble

## Maturity assessment

- [ ] minimal bugs and code debt
- [ ] rich feature set
- [ ] UX
	- [ ] well reasoned
	- [ ] self-consistent
	- [ ] familiar look and feel
- [ ] integration with other related pieces
- [ ] saves time
- [ ] fulfills an obvious and strong need
- [ ] general purpose
- [ ] awesome customer support
