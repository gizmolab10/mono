## Problem

We want to paint each SO a different color. The painter's algorithm can't do this — it paints whole faces back to front, so when two SOs intersect, the last one wins and looks bad.

Facets solve this: each facet is a region that's entirely on one face, entirely visible. Paint each facet with its SO's color — a 2D mosaic.

## Data

All visible line segments come from the topology pipeline. Only front-facing faces are processed.

- **edge segments** — each SO edge, clipped for occlusion
- **intersection lines** — where two SO faces cross in 3D, clipped to both face quads and for occlusion
- **occluding segments** — edge passing in front of another SO's face, forming a boundary of the visible region

Each segment knows: which SO, which face, edge or intersection type, and its two endpoint keys.

## Endpoint types

An endpoint is where line segments meet. The type depends on what caused the point:

- **corner** — a mesh vertex. Key: SO + vertex index.
- **face intersection (fi)** — where an intersection line hits a face edge. Key: both face IDs + start/end.
- **occlusion clip (oc)** — where an occluder hides part of an edge. Key: edge + occluder face + enter/exit.
- **edge crossing (ex)** — two edges from different SOs cross on screen. Key: both edge IDs.

Each endpoint is uniquely identified by the geometric features that create it — no coordinates.

## Tracing

1. Choose a face, collect its segments.
2. Start at a 90-degree corner. Two segments point outward — go left.
3. At each endpoint, follow the next segment clockwise (cyclic ordering).
4. Repeat until home (facet) or lost (dud).

## Filtering

Not all traced polygons should be painted. For each polygon: is it on the selected SO's visible face, or behind another SO? Test by checking depth against occluding faces (other SOs only — self-occlusion can't happen for convex SOs).

## Cyclic ordering

At each endpoint, segments radiate like spokes. Project each segment's direction onto the face's tangent plane, compute the angle, sort by angle. This ordering is view-independent — computed once per face.
