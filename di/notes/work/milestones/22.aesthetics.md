## separator

the separator design is below, but i want the flares to be an svg

plain lines feel dead. i wanted separators that breathe — that feel like they belong to the surfaces they divide, not just painted on top. so every separator grows little rounded flares at each end, colored `--bg`, that carve into the accent-colored bar. looks like the bar emerges from behind the content. subtle, but it makes everything feel joined rather than stacked.

### anatomy

a separator is a `<button>` (for optional click handling) with three layers:

1. **bar** — the accent-colored line itself. thickness is a prop, defaults to `k.thickness.separator.content` (~3.3px). horizontal separators stretch full width; vertical ones use `align-self: stretch` to fill their flex parent (or a fixed `length` if provided).

2. **flares** — two `<div>`s positioned at opposite ends, outside the bar. radius = `thickness × 3.2`. they're `--bg`-colored with one-sided border-radius, so they look like rounded notches cut into the bar's ends. a `::before` pseudo-element extends the accent color behind the flares so the bar doesn't have gaps.

3. **title** (optional) — absolutely centered text, `--h-font-small`, 60% opacity (full on hover when clickable).

### props

| prop | default | what it does |
| ---- | ------- | ------------ |
| `vertical` | `false` | orientation — flips layout, margin direction, flare placement |
| `thickness` | `k.thickness.separator.content` | bar width (vertical) or height (horizontal) |
| `length` | `0` | fixed height for vertical seps. `0` means stretch to parent |
| `margin` | `11` | how far the bar extends past its container. negative margin pulls the bar outward. `end` adds extra on one side |
| `end` | `0` | additional extension on the bottom (vertical) or right (horizontal) end |
| `title` | — | centered label |
| `onclick` | — | makes it clickable via the hits system, adds pointer cursor |

### margin trick

the key trick: horizontal separators use `margin: 4px calc(-1 * var(--e)) 4px calc(-1 * var(--m))` and `width: calc(100% + var(--m) + var(--e))`. the negative margins pull the bar outside its parent's padding, so it bleeds edge-to-edge. vertical separators do the same thing rotated 90° — negative top/bottom margins let them extend past their row.

### hit system

when `onclick` is provided, the separator registers as an `S_Hit_Target` (type: control) with a random uid. mouse-up triggers the callback. cleanup on unmount via `hits.delete_hit_target`.

### where it's used

- **controls** — vertical seps between button groups (thickness: `separator.main`, ~6.6px). horizontal seps between wrapped rows. the ham↔other separator uses `margin={7}` to extend upward into the controls bar edge.
- **details panels** — default horizontal seps (thickness: `separator.content`, ~3.3px) between sections in preferences, parts, and library.
