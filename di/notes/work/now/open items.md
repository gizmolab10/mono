- **Coordinate-frame mixing audit.** Standing item, fires on the next coordinate-frame bug (a place where a number in one reference frame is multiplied or compared with a number in another, with no conversion). Three escalating levels:
    1. Quick grep pass (10-30 minutes). Scan the placement file, the renderer, and the helpers for places that multiply or compare variables in different reference frames. Things to look for: an edge-length number times a dim-line-length number, a world-3D vector alongside a screen-2D vector with no conversion, an averaged witness direction used in one line and per-endpoint witness directions in the next.
    2. Manual code walk (a couple of hours). Trace every coordinate-using path end to end, document what frame each variable is in (edge, dim line, world, NDC), check every multiplication and comparison for frame consistency.
    3. Branded TypeScript types (a few hours of refactor). Types like EdgeSpacePx and DimLineSpacePx so the compiler refuses to mix them silently. Permanent guard, touches many files.

    On the next surface, do level 1 first. Escalate to level 2 only if more than one suspicious spot turns up. Level 3 is reserved for a separate refactor session. *Effort: trigger-driven; level 1 ten to thirty minutes.*
